### ✅ 회원가입 기능

| 기능 ID | **F-001** |
| --- | --- |
| 기능명 | 회원가입 (User Register) |
| 설명 | 사용자가 이름, 아이디, 비밀번호, 이메일을 입력하여 회원으로 등록한다. |
| 연관 UC | UC-001 (회원가입) |
| HTTP 메서드 | POST |
| 요청 URL | `/member/register` |
| 입력값 | JSON 또는 FormData: `{ id, pw, name, email }` |
| 출력값 | 성공 메시지 / 실패 시 에러 메시지 |

**처리 흐름**

1. 클라이언트에서 가입 폼 작성
2. 서버에 POST 요청 전송
3. 중복 확인 및 입력값 검증
4. 비밀번호 암호화
5. DB에 저장
6. 성공 시 로그인 페이지로 이동 또는 성공 메시지 반환 

### ✅ 로그인 기능

| 기능 ID | **F-002** |
| --- | --- |
| 기능명 | 로그인 (User Login) |
| 설명 | 사용자가 아이디와 비밀번호를 입력하여 인증을 진행하고 세션을 생성한다. |
| 연관 UC | UC-002 (로그인) |
| HTTP 메서드 | POST |
| 요청 URL | `/member/login` |
| 입력값 | `{ id, pw }` |
| 출력값 | 로그인 성공 여부 및 리다이렉션 URL |

**처리 흐름**

1. 사용자가 로그인 페이지에서 입력
2. 서버에서 아이디/비밀번호 일치 확인
3. 세션 생성 (Spring Session)
4. 메인 페이지 또는 마이페이지로 리다이렉션 |

### ✅ 상품 상세조회 기능

| 기능 ID | F-003 |
| --- | --- |
| 기능명 | 상품 상세 조회 |
| 설명 | 특정 상품의 상세 정보를 보여준다. |
| 연관 UC | UC-003 (상품 상세보기) |
| HTTP 메서드 | GET |
| 요청 URL | `/product/detail/{no}` |
| 입력값 | `상품 번호 (PathVariable)` |
| 출력값 | 상품 객체(JSON 또는 Model 데이터) |

**처리 흐름**

1. 사용자가 목록에서 상품 클릭
2. 서버는 해당 상품 번호로 DB에서 조회
3. 조회수 증가 처리
4. 결과를 페이지 또는 JSON으로 응답 |
