## 2. 도메인별 요청 처리 흐름

### 2-1. 회원 인증 흐름 (회원가입 / 로그인 / 로그아웃)

```mermaid
flowchart TD
    A([클라이언트 요청]) --> B{요청 유형}

    B -->|POST /signup| C[AuthController.signup]
    C --> D[MemberService.signup]
    D --> E{이메일 중복 확인\nMemberRepository.existsByEmail}
    E -->|중복| F[409 Conflict 반환]
    E -->|사용 가능| G[BCrypt 비밀번호 암호화]
    G --> H[MemberRepository.save]
    H --> I[(member 테이블 INSERT)]
    I --> J[201 Created 반환]

    B -->|POST /signin| K[AuthController.signin]
    K --> L[MemberService.signin]
    L --> M[MemberRepository.findByEmail]
    M --> N{회원 존재 여부}
    N -->|없음| O[401 Unauthorized 반환]
    N -->|있음| P{비밀번호 검증\nBCrypt.matches}
    P -->|불일치| O
    P -->|일치| Q{MemberStatus 확인}
    Q -->|DELETE / DEACTIVATE| R[403 Forbidden 반환]
    Q -->|ACTIVATE| S[JwtUtil.generateAccessToken]
    S --> T[JwtUtil.generateRefreshToken]
    T --> U[200 OK + AccessToken + RefreshToken 반환]

    B -->|POST /signout| V[AuthController.signout]
    V --> W[MemberService.signout]
    W --> X[RefreshToken 무효화]
    X --> Y[200 OK 반환]
```
