## 🚨 D-4 우선순위 작업 목록

### 🔴 필수 (데모/채점에 영향)

1. **배송 자동 생성** — `PaymentService.confirmToss()` / `approvePayment()` 완료 후 `Delivery` 레코드 생성 추가 (REQ-D-001)
2. **상품 키워드 검색** — `ProductRepository`에 `findByTitleContainingOrAuthorContaining()` 추가 + `GET /api/product?keyword=` (REQ-P-003)
3. **대시보드 보완** — `DashboardResponse`에 미답변QnA 수, 재고부족 상품 수, 오늘 매출, 이번달 매출, 신규 회원 수 필드 추가 (REQ-A-002, A-003, A-006, A-007)
4. **마이페이지 리뷰 목록 / 삭제** — `GET /api/member/me/reviews`, `DELETE /api/review/{id}` 마이페이지 연동 (REQ-M-016, M-019)
5. **마이페이지 QnA 목록 / 삭제** — `GET /api/member/me/qnas` 추가 (REQ-M-021)

### 🟠 권장 (기능 완성도)

6. **리뷰 수정** — `PUT /api/review/{reviewId}` + ReviewService.updateReview() (REQ-M-018)
7. **QnA 수정** — `PUT /api/qna/{qnaId}` + QnAService.updateQnA() (REQ-M-023)
8. **Cart is_checked 토글 API** — `PATCH /api/cart/{cartItemId}/check` (REQ-C-004)
9. **관리자 공지/QnA/리뷰 상세 조회** — 기존 user API 재사용 또는 신규 추가 (FA-031, FA-036, FA-039)
10. **OAuth provider 불일치 예외처리** — OAuthMemberService에 동일 이메일 다른 provider 체크 (REQ-AUTH-005)

### 🟢 여유 시 (낮은 우선순위)

11. QnA n차 재문의 (depth 구조) — 엔티티 변경 수반으로 리스크 있음
12. QnA 상태 ENUM (WAITING/PROCESSING/COMPLETE) — answer 필드 기반 현재 구조 유지 가능
13. 리뷰 이미지 첨부 (ReviewImage 테이블)
14. 리뷰 반응 (좋아요/싫어요)
15. 관리자 카테고리 CRUD
