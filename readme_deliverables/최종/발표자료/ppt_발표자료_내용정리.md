# 온라인 서점 웹 서비스 프로젝트 발표자료

## 1. 표지

### 슬라이드 제목
온라인 서점 웹 서비스 프로젝트

### 부제
Vue + Spring Boot + PostgreSQL 기반 통합 쇼핑 플랫폼

### 표기 정보
- 팀 프로젝트 발표 자료
- 팀명
- 발표자명
- 발표일

### 발표 멘트
안녕하세요. 저희는 Vue와 Spring Boot, PostgreSQL을 기반으로 온라인 서점 웹 서비스를 구현했습니다. 이번 발표에서는 프로젝트의 기획 배경부터 시스템 구조, 핵심 기능, 그리고 구현 포인트까지 설명드리겠습니다.

## 2. 프로젝트 소개

### 슬라이드 제목
프로젝트 소개

### 핵심 내용
- 도서 검색, 주문, 결제, 배송, 리뷰, QnA를 통합한 온라인 서점 서비스
- 일반 사용자와 관리자 페이지를 분리한 실서비스형 웹 프로젝트
- 개발 기간: 2026.03.13 ~ 2026.04.12
- 단순 CRUD를 넘어 서비스 흐름 전체를 구현하는 것을 목표로 진행

### 발표 멘트
이 프로젝트는 사용자가 도서를 검색하고 구매하며, 리뷰와 QnA를 작성할 수 있고, 관리자는 상품과 주문, 회원, 게시판을 운영할 수 있는 온라인 서점 서비스입니다. 단순 조회형 사이트가 아니라 실제 서비스 흐름을 고려해 설계하고 구현한 것이 핵심입니다.

## 3. 개발 배경 및 목적

### 슬라이드 제목
개발 배경 및 목적

### 핵심 내용
- 실무에 가까운 전형적인 커머스 구조를 직접 설계하고 구현
- 회원 인증, 상품 관리, 주문, 결제, 배송까지 하나의 흐름으로 연결

### 발표 멘트
저희는 단순 CRUD를 넘어서 실제 서비스 형태에 가까운 프로젝트를 만들고자 했습니다. 그래서 회원 인증, 상품 관리, 주문과 결제, 배송, 관리자 기능까지 하나의 흐름으로 연결되는 구조를 목표로 삼았습니다. 이를 통해 프론트엔드와 백엔드, DB, 인프라를 통합적으로 경험하고자 했습니다.

주요 사용자와 요구사항

## 5. 기술 스택

### 슬라이드 제목
기술 스택

### 핵심 내용
- Frontend: Vue 3, Vue Router, Pinia, Axios, Tailwind CSS
- Backend: Java 21, Spring Boot 3, Spring Security, JPA
- Database: PostgreSQL
- Infra: Docker, Nginx
- Auth/Payment: JWT, OAuth, Payment Gateway 구조

### 발표 멘트
프론트엔드는 Vue 3 기반 SPA로 구성했고, 상태 관리는 Pinia, API 통신은 Axios를 사용했습니다. 백엔드는 Spring Boot와 JPA를 기반으로 REST API를 구성했으며, 인증과 권한 처리는 Spring Security와 JWT를 사용했습니다. 데이터베이스는 PostgreSQL이고, 배포와 실행 환경은 Docker와 Nginx를 고려해 구성했습니다.

## 6. 시스템 아키텍처

### 슬라이드 제목
시스템 아키텍처

### 핵심 내용
- Vue 클라이언트가 Spring Boot REST API와 통신하는 SPA 구조
- 백엔드에서 인증, 권한, 비즈니스 로직, 데이터 접근을 담당
- PostgreSQL에 회원, 상품, 주문, 결제, 배송, 게시판 데이터 저장
- JWT 기반 인증으로 Stateless 구조 유지
- 관리자와 일반 사용자 기능을 라우팅 및 권한 레벨에서 분리

### 발표 멘트
전체 구조는 Vue 클라이언트가 Spring Boot REST API와 통신하고, 백엔드가 PostgreSQL과 연동하는 형태입니다. 인증은 JWT 기반으로 처리했고, 관리자와 일반 사용자 기능을 라우팅과 권한 처리 단계에서 분리했습니다. 이를 통해 보안성과 유지보수성을 함께 고려했습니다.

## 7. 핵심 기능 소개

### 슬라이드 제목
핵심 기능 소개

### 핵심 내용
- 회원가입, 로그인, OAuth 로그인, JWT 인증
- 상품 목록, 상세 조회, 검색, 카테고리 필터
- 장바구니, 주문 생성, 결제 요청 및 완료 처리

- 관리자 대시보드와 운영 관리 기능 제공

### 발표 멘트
핵심 기능은 크게 다섯 가지입니다. 첫째, 회원가입과 로그인, OAuth 기반 인증 기능입니다. 둘째, 상품 목록과 상세 조회, 검색 기능입니다. 셋째, 장바구니부터 주문, 결제까지 이어지는 구매 흐름입니다. 넷째, 마이페이지에서 주문 내역과 리뷰, QnA를 관리할 수 있습니다. 마지막으로 관리자는 전체 운영 기능을 수행할 수 있습니다.

---

## 8. 인증 및 권한 처리

### 슬라이드 제목
인증 및 권한 처리

### 핵심 내용
- JWT 기반 Stateless 인증 구조 적용
- Access Token + Refresh Token 발급 및 갱신 흐름 설계
- Google / Kakao OAuth 로그인 확장 반영
- Spring Security로 API 접근 제어
- ADMIN, MANAGER, USER 역할에 따라 기능 제한

### 발표 멘트
인증은 JWT 기반 Stateless 방식으로 구현했습니다. 로그인 시 Access Token과 Refresh Token을 발급하고, 토큰 만료 시 갱신 흐름도 고려했습니다. 또한 Google, Kakao OAuth 로그인 확장도 반영했습니다. 관리자 페이지는 별도의 권한 검증을 통해 일반 사용자와 명확히 구분되도록 설계했습니다.

---

## 9. 주문 · 결제 · 배송 흐름

### 슬라이드 제목
주문 · 결제 · 배송 흐름

### 핵심 내용
- 장바구니 선택 상품 기준으로 주문 생성
- 결제 성공 시 payment 상태 갱신 후 delivery 자동 생성
- 주문, 결제, 배송 상태를 각각 분리해 일관성 있게 관리
- 결제 실패, 주문 취소, 환불 시나리오까지 고려
- 다중 PG 확장을 위해 Gateway 방식 적용

### 발표 멘트
이 프로젝트에서 가장 중요한 흐름은 주문, 결제, 배송입니다. 사용자가 장바구니에서 상품을 선택하면 주문이 생성되고, 결제가 완료되면 배송 정보가 생성됩니다. 이 과정에서 주문 상태, 결제 상태, 배송 상태를 분리해서 관리했고, 결제 실패나 취소 같은 예외 상황도 함께 처리할 수 있도록 설계했습니다.

## 10. 관리자 기능

### 슬라이드 제목
관리자 기능

### 핵심 내용
- 대시보드에서 주문, QnA, 재고 부족 현황 확인
- 상품 및 카테고리 등록, 수정, 삭제
- 회원 목록 조회 및 권한/상태 관리
- 주문 승인, 결제 내역 조회, 배송 상태 변경
- 공지사항, 리뷰, QnA 운영 관리

### 발표 멘트
관리자 페이지는 실제 운영 기능에 가깝게 구성했습니다. 대시보드에서 전체 현황을 확인할 수 있고, 상품과 카테고리 관리, 회원 관리, 주문과 배송 처리, 그리고 공지사항, 리뷰, QnA 관리까지 가능합니다. 단순히 데이터만 보는 것이 아니라 실제 운영 관점에서 필요한 기능 중심으로 설계했습니다.

## 11. 구현 중 어려움과 해결

### 슬라이드 제목
구현 중 어려움과 해결

결론 및 시연 안내
